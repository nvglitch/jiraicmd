## 资源差异

网页初稿里有:

- `photo\huaban-5891557590.png`

当前游戏工程的 `jiraicmd\photo` 目录里没有这个 png 原图，但 Godot 侧 `main_terminal.gd` 仍然引用了:

```gdscript
"res://jiraicmd/photo/huaban-5891557590.png"
```

如果游戏里第一张图依赖这个资源，建议把网页初稿里的 `photo\huaban-5891557590.png` 补回到:

```text
D:\downloads\jirai-terminal\jiraicmd\photo\huaban-5891557590.png
```

然后在 Godot 编辑器中重新导入，生成对应 `.import` 文件。

## Godot 版相比网页版的主要新增/改造

当前游戏工程把网页交互移植成 Godot UI，主要差异包括：

- 用 `main_terminal.gd` 实现终端命令、主题、自动补全、gallery、maimai/chuni 排行、yandere 彩蛋。
- 用 `gallery.tscn` 实现图片预览弹窗。
- 用 `char_pink.png` / `char_mizuiro.png` 实现主题角色立绘切换。
- 用 `user://jirai_terminal.cfg` 保存主题配置。
- 用 `maimai_data.json` / `chuni_data.json` 给 Godot 读取数据，网页仍使用 `.js` 数据文件。
- gallery 选择光标在 Godot 里也已改为跟随主题的反白高亮条。

这些功能大多已经有网页版等价实现；如果未来要继续同步，建议从功能行为对照，而不是直接复制 Godot 脚本。

## 建议同步顺序

1. 保持网页初稿的 `index.html` 作为网页主线。
2. 每次游戏工程里的 `jiraicmd\index.html` 有变化时，先用 diff 对比两个网页文件。
3. 只把网页相关改动同步回来，不复制 Godot `.tscn`、`.gd`、`.import` 文件。
4. 如果游戏工程新增网页资源，优先同步这些类型：
   - `.html`
   - `.js`
   - `.css`
   - `.csv`
   - `photo` 目录里的图片
   - 页面直接引用的 `.png` / `.jpg`
5. 如果游戏工程新增 Godot-only 资源，先确认网页是否真的需要，再决定是否复制。
